Developing for the GBA: Part 1

Today, I'll be teaching you how to write ROMs for the GameBoy Advance.

Installation

The github repository for my development suite is at https://github.com/adiprad/gba-development.

The README for the repo should explain how to install it on Windows and OSX.

Setup and First GBA Program!

To setup a new GBA Project, simply clone the gba_template folder where you want to setup your project.

The file structure should now look like this: 

+-- build.bat
+-- m4.h
+-- main.c

Let me explain:

build.bat is a batch file that will compile the main.c file into a GBA ROM (assuming you installed it correctly). You can edit this file based on what .c files you are including

m4.h is a pretty cool header file that makes it easier to render graphics on the GBA.

main.c is the .c file that is the entry point of your program. We will be writing our code in this file (obviously).

So, lets get to writing our GBA rom.

This rom won't be doing anything interesting (just a blank screen).

Write the following into main.c: 

#include "m4.h"

int main() {
    m4_init();
    return 0;
}

Save the file, and run build.bat.

Assuming everything went fine, run the ROM in your friendly neighborhood emulator.

What emulator should I use, you ask?

I personally recommend VBA-M, which is probably the best GBA Emulator out there.

You can download it from the Chrome App Store, or from the site at http://vba-m.com/index.php?ind=downloads

When you run the ROM in you newfound emulator, it should show a black screen.

If the ROM works, then Congratulations!

If it doesn't, then try reinstalling the repository.
